# QA Engineer Task Templates

## 1. Точечная проверка после фикса
Title: `Проверить фикс <краткое название>`

Description:
`Проверь исправление для <страница/функция>. Сначала воспроизведи основной сценарий, затем проверь ближайшие регрессии, консоль, сеть и состояния ошибки/пустых данных, если они релевантны. Код продукта не меняй. Нужен отчет: что проверено, что прошло, что сломано, что осталось непроверенным.`

## 2. Глубокий аудит одного интерфейса
Title: `Провести QA-аудит <страница/раздел>`

Description:
`Проведи аудит страницы или раздела <страница/раздел>. Проверь рендер, клики, ссылки, формы, responsive-поведение, консоль, сеть, пустые и ошибочные состояния. Если runtime недоступен, сделай статический аудит по коду и явно укажи ограничения. Код продукта не меняй без явного запроса.`

## 3. Проверка после серии исправлений
Title: `Проверить регрессии после правок в <область>`

Description:
`Проверь область <область> после последних исправлений. Нужен не только happy path, но и поиск побочных эффектов: сломанные ссылки, несоответствие текста, ошибки API, визуальные дефекты, проблемы с состоянием, edge cases. Верни краткий отчет с severity по найденным проблемам.`
