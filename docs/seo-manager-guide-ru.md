# SEO-кабинет: инструкция для контент-менеджера

Эта инструкция нужна, чтобы страницы из SEO-кабинета корректно показывались на сайте.

## 1) Что важно знать перед началом

- **Страница услуги (встроенный URL)** будет заменена SEO-контентом **только если выбран `Layout: create-page`**.
- Если `Layout` не выбран (`none`), встроенная страница остается в стандартном дизайне сайта.
- Для новых URL (которых нет среди встроенных страниц) страница открывается как обычная SEO-страница.
- Публикация работает только при `Опубликовано = включено`.

## 2) Быстрый сценарий: заменить встроенную страницу услуги

1. Откройте SEO-кабинет и нажмите **Создать страницу** (или откройте существующую).
2. В поле **URL Path (Slug)** укажите нужный slug, например `bankovskie-garantii` (без `/`).
3. В блоке **Шаблоны страницы** выберите:
   - **Layout шаблон**: `Layout: create-page`
   - (Опционально) **Контентный пресет** для автозаполнения.
4. Заполните вкладки:
   - **Мета-теги**
   - **Контент**
   - **FAQ**
   - **Предложения**
5. Включите переключатель **Опубликовано**.
6. Нажмите **Сохранить**.
7. Проверьте страницу на сайте в режиме инкогнито.

## 3) Быстрый сценарий: создать новую SEO-страницу (новый URL)

1. Укажите уникальный **Slug**, которого нет среди встроенных маршрутов.
2. Если нужен шаблонный лендинг-вид, выберите `Layout: create-page`.
3. Заполните контент и мета-поля.
4. Включите **Опубликовано** и сохраните.
5. Проверьте URL напрямую: `https://ваш-домен/<slug>`.

## 4) Как заполнять поля правильно

## Основное

- **URL Path (Slug)**: только путь, без домена и без первого `/`.
  - Хорошо: `bankovskie-garantii`
  - Плохо: `/bankovskie-garantii`, `https://site.ru/page`
- **Тип страницы**:
  - `product` - для услуг
  - `custom` - для нестандартных страниц
- **Layout шаблон**:
  - `create-page` - включает шаблонный рендер SEO-страницы
  - `none` - шаблонный рендер выключен
- **Опубликовано**: страница доступна публично только при включенном флаге.

## Мета-теги

- **Meta title**: 50-70 символов, ключевая услуга ближе к началу.
- **Meta description**: 120-170 символов, понятное УТП и действие.
- **Meta keywords**: опционально, перечисление через запятую.

## Контент

- **H1**: один главный заголовок страницы.
- **Описание**: главное коммерческое предложение + выгоды.
- **Кнопка hero**:
  - Текст: краткий CTA (например, `Оставить заявку`)
  - Ссылка: `#application`, `/slug` или `https://...`
- **Часто ищут**: каждая строка = текст + ссылка.

## FAQ

- Ответы короткие, предметные, без воды.

## 5) Что чаще всего ломает отображение

- Выбрали нужный slug, но **не выбрали `Layout: create-page`** - встроенная страница не заменится.
- Сохранили страницу как черновик (**Опубликовано выключено**).
- В ссылках указали некорректный формат.
  - Разрешено: `#якорь`, `/локальный-путь`, `https://внешний-сайт`
- Изменили не тот slug (опечатка, другой дефис, старая транслитерация).

## 6) Проверка после публикации (чеклист)

- Открывается нужный URL.
- Виден новый H1 и текст из SEO-кабинета.
- Кнопка hero ведет в правильное место.
- Блок «Часто ищут» кликабелен и без битых ссылок.
- Title/Description обновились (проверить в исходном коде страницы).

## 7) Как откатить изменения быстро

Вариант A (мягкий):
- Отключите **Опубликовано** и сохраните.

Вариант B (для встроенной услуги):
- Оставьте страницу опубликованной, но поставьте `Layout = none`.

## 8) Рекомендованный рабочий процесс для заказчика

1. Сделать черновик и заполнить все вкладки.
2. Отдать на быстрый просмотр ответственному.
3. Включить `Опубликовано`.
4. Проверить URL в инкогнито.
5. Добавить внутренние ссылки через блок «Часто ищут».

Если нужна, можно сделать отдельный короткий PDF-чеклист для менеджеров (1 страница) на основе этой инструкции.
